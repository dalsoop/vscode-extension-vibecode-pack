# Vibecode Right-Click — `.sh` Actions

> **`.sh` 파일 우클릭 → 통합 터미널에서 바로 실행.**
> dirty 상태면 자동 저장. 경로 입력 0.

---

## 누구를 위한 거?

- `bash ./deploy.sh` 같은 셸 스크립트를 자주 돌리는 인프라 / DevOps
- 빌드 자동화 스크립트가 많은 모노레포에서 일하는 개발자
- 에디터로 스크립트 편집 → 저장 → 터미널 가서 실행 의 반복이 지긋지긋한 사람
- 신입에게 "이 폴더의 `.sh` 누르고 ▶ 만 누르세요" 라고 안내하고 싶은 리드

## 풀어주는 문제

| 평소 흔한 상황 | 이 확장으로 |
|---|---|
| 에디터로 `.sh` 수정 → 저장 → 터미널 → cd → `bash ./script.sh` | 우클릭 한 번. 그게 끝. |
| 저장 안 하고 실행하면 옛날 버전 돈다 | **자동 저장 후 실행** — 항상 최신 버전 |
| 어느 위치에서 실행했는지 헷갈림 | 통합 터미널을 **스크립트가 있는 폴더로 cd 후** 실행 |
| 명령 팔레트로 실행하고 싶음 | 활성 에디터 기준으로 팔레트에서도 실행 가능 |
| 에디터에서 빠르게 다시 돌리고 싶음 | `.sh` 파일 에디터 우상단 **▶ 플레이 버튼** |

## 핵심 셀링 포인트

1. **3가지 트리거** — 우클릭 (Explorer), 에디터 본문 우클릭, ▶ 플레이 버튼.
2. **dirty 자동 저장** — 편집 중인 스크립트라도 실행 직전에 저장. "어 이상한데?" 라고 헤매지 않음.
3. **올바른 cwd 보장** — 통합 터미널을 스크립트 위치에 열어서 상대 경로가 깨지지 않음.
4. **single-quote escape** — 파일명에 공백·특수문자 있어도 안전 (`bash '<quoted-path>'`).
5. **팔레트 지원** — 활성 에디터의 `.sh` 를 키보드만으로 실행.

## 사용 흐름 (시각)

```
Explorer
  ▸ scripts/
    ■ deploy.sh   ◄── 우클릭
         │
         └─ "Run .sh in Integrated Terminal"
                          ↓
              ┌───────── Terminal ─────────┐
              │ ~/repo/scripts $           │
              │ $ bash './deploy.sh'       │
              │ → Deploying...             │
              │ → Done.                    │
              └────────────────────────────┘


.sh 파일을 에디터로 열었다면:
      ┌─ deploy.sh ──────────  ▶ ◄── 플레이 버튼
      │ #!/usr/bin/env bash
      │ set -euo pipefail
      │ ...
      └────────────────────────────
```

## 썸네일 아이디어

- **메인 컷**: `.sh` 파일 아이콘 옆에 큰 ▶ 플레이 버튼 — 그 화살표가 터미널 창으로 이어지는 구도
- **포인트**: 터미널 안에 `$ bash './deploy.sh'` 라인 + 그 아래 초록색 출력 로그
- **컬러 키**: 다크 + 그린 (실행 OK) + 터미널 모노폰트
- **부제**: "Right-click · Auto-save · Run"
- **사이드 디테일**: 에디터 우상단 ▶ 아이콘 / Explorer 우클릭 메뉴 미니어처를 한 컷에

## 한 줄 후크

> "스크립트는 클릭 한 번, 저장은 알아서."
> 매번 터미널 cd 할 필요 없다.
