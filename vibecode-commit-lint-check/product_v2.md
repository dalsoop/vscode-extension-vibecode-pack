# Product V2 - Vibecode Commit Lint Check

> 셀링 전용 문서. 개발 README가 아니라 마켓플레이스 설명, 썸네일 기획, 소개 이미지 제작에 쓰는 메시지다.

## 한 줄 판매 문구

커밋 메시지를 Conventional, Node, Python, Laravel 템플릿으로 검사하고 팀 규칙을 폴더에 바로 설치한다.

## 어떤 목적으로 쓰나

커밋 메시지 품질을 릴리즈 노트, changelog, CI 규칙과 맞추기 위해 쓴다. 처음에는 내장 검사기로 바로 시작하고, 필요해지면 commitlint 설정이나 `.vibecode/code-lint` 셸 체크로 강도를 올리는 구조다.

## 누구에게 팔아야 하나

- 커밋 메시지 규칙을 팀에 정착시키려는 리드
- Conventional Commits 기반 자동 릴리즈를 쓰는 팀
- 레포마다 commitlint 세팅을 반복 적용해야 하는 개발자

## 사용법 시나리오

1. Commit Lint 사이드바를 연다.
2. 번들 템플릿에서 프로젝트에 맞는 규칙을 고른다.
3. 폴더 우클릭 또는 트리 액션으로 설정을 스캐폴드한다.
4. 마지막 커밋을 검사한다.
5. Checks 트리에서 추가 셸 체크를 실행하고 결과를 확인한다.

## 썸네일 제작 지시

- 실제 그릴 장면: VSCode 사이드바에 Checks와 Templates 트리가 있고, 가운데에는 커밋 메시지 카드가 초록 체크/빨간 경고로 판정되는 장면.
- 반드시 보여야 할 목적: "커밋 메시지를 규칙으로 검사한다"가 보여야 한다.
- 시각 요소: `feat: add login` 같은 커밋 제목, 체크 결과, 템플릿 목록, 터미널 검사 출력.
- 피할 것: 체크마크 아이콘 하나만 크게 그리기, git 로고만 있는 썸네일, 추상 보안 이미지.

## 썸네일 카피

`Commit rules before push`
