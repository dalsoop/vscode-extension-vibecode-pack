# Vibecode Extension Menu List

> **상태바의 `▶ Vibecode` 버튼 클릭 → 설치된 모든 vibecode-* 확장의 명령을 카드로 한 눈에.**
> 어떤 확장에 어떤 명령이 있더라? 그걸 더 이상 외우지 않아도 된다.

---

## 누구를 위한 거?

- vibecode-* 확장을 5개 이상 깔고 쓰는 사람 — 명령어 다 기억 못 함
- 다른 publisher 가 만든 vibecode-* 확장도 통합해서 보고 싶은 팀
- 명령 팔레트 (`⌘⇧P`) → 타이핑보다 **카드 그리드 + 클릭** 이 편한 사람
- 사내 신규 입사자에게 "우리 팀이 쓰는 vibecode 확장 모음" 을 한 화면으로 보여주고 싶은 리드

## 풀어주는 문제

| 평소 흔한 상황 | 이 확장으로 |
|---|---|
| 확장이 늘어날수록 명령어 이름 외우기 부담 | **카드 그리드** — 확장별 섹션 + 각 명령의 제목·실제 commandId |
| 팔레트 검색은 영문 정확히 알아야 함 | **상단 필터** 에 타이핑하면 즉시 매칭 (제목/카테고리/commandId/확장명) |
| 신규 확장 깔았는데 어디 있는지 못 찾음 | `vscode.extensions.onDidChange` 구독 — 설치/제거 시 패널 즉시 갱신 |
| 다른 publisher 의 vibecode-* 확장은 안 보임 | **publisher 무관**. 확장 id 의 name 이 `vibecode-` 로 시작하면 자동 노출 |
| 키보드 워크플로우만 쓰고 싶음 | 팔레트에서 `Show Extension Menu Catalog (QuickPick)` 도 제공 |

## 핵심 셀링 포인트

1. **상태바 1버튼** — 우측 `▶ Vibecode` 클릭 → `ViewColumn.Beside` 로 슬라이드인.
2. **자동 디스커버리** — `publisher` 무관, 확장 name 의 `vibecode-` 프리픽스만 보고 잡음. 등록·협의 0.
3. **NLS 자동 해석** — 라벨이 `%nls.key%` 면 해당 확장의 `package.nls.<locale>.json` 을 직접 읽어 한국어로도 표시.
4. **자동 갱신** — 다른 확장 설치/제거하면 패널이 즉시 다시 그림.
5. **`retainContextWhenHidden`** — 다른 탭 갔다 와도 스크롤·필터 상태 유지.
6. **3가지 트리거** — 상태바 / 팔레트 / QuickPick. 손/키보드 둘 다.
7. **VSCode 테마 자동 적응** — 다크/라이트/하이콘트라스트 다 OK.

## 썸네일 아이디어

- **메인 컷**: VSCode 우측 사이드패널이 슬라이드인하는 모습 — 안에 카드 그리드가 3-4 섹션으로 분류
- **상단**: 검색창 `⌕ commit` 입력하면 매칭 카드만 남는 필터링 모션
- **하단 강조**: VSCode 상태바 우측의 `▶ Vibecode` 칩 버튼이 펄스 — 거기서 패널이 열리는 화살표
- **컬러 키**: VSCode 다크 배경 + 카드는 살짝 elevated 그레이, 섹션 헤더는 시안
- **부제**: "All vibecode commands · One panel"
