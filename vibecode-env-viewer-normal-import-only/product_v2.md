# Product V2 - Vibecode .env Import-Only

> 셀링 전용 문서. 개발 README가 아니라 마켓플레이스 설명, 썸네일 기획, 소개 이미지 제작에 쓰는 메시지다.

## 한 줄 판매 문구

`.env` 키는 보여주고 값은 절대 표시하지 않는 붙여넣기 전용 보안 뷰.

## 어떤 목적으로 쓰나

화면 공유, 녹화, 옆자리 노출 상황에서도 `.env` 값을 실수로 보여주지 않게 만든다. 값 입력은 붙여넣기로만 허용하고 DOM에 남기지 않아, secret을 다룰 때 평소보다 안전한 편집 흐름을 제공한다.

## 누구에게 팔아야 하나

- 화면 공유 중 `.env`를 열어야 하는 개발자
- 신입이나 외주 작업자에게 값 노출 없이 키 목록만 확인시켜야 하는 팀
- `.env.example`과 실제 `.env`의 누락 키를 빠르게 맞추고 싶은 사람

## 사용법 시나리오

1. `.env` 파일을 열면 import-only 커스텀 에디터가 열린다.
2. 키 이름과 설정 여부만 확인한다.
3. 값 칸에는 직접 타이핑하지 않고 붙여넣기만 한다.
4. 저장하면 기존 `.env` 형식으로 기록된다.
5. `.env.example`과 비교해 누락 키를 추가한다.

## 썸네일 제작 지시

- 실제 그릴 장면: VSCode 안 `.env` 편집 화면에서 키 목록은 선명하게 보이고 값 영역은 `... (set)` 같은 마스크로 가려진 장면.
- 반드시 보여야 할 목적: "값을 보여주지 않고 `.env`를 편집한다"가 보여야 한다.
- 시각 요소: `STRIPE_SECRET`, `OPENAI_API_KEY` 같은 키 이름, 빈/설정됨 상태 배지, 붙여넣기 전용 입력칸, 화면 공유 느낌의 오버레이.
- 피할 것: 보안 방패 아이콘만 있는 이미지, 실제 키/값 UI가 없는 추상 이미지.
