# Product V2 - Vibecode .env Import-Only Encrypted

> 셀링 전용 문서. 개발 README가 아니라 마켓플레이스 설명, 썸네일 기획, 소개 이미지 제작에 쓰는 메시지다.

## 한 줄 판매 문구

`.env` 값은 붙여넣기로만 입력하고, 디스크에는 암호화된 값만 저장하는 보안 에디터.

## 어떤 목적으로 쓰나

환경변수 값을 화면, 클립보드, 디스크에 평문으로 노출하는 위험을 줄인다. 키 이름은 보여주되 값은 마스킹하고, 저장 시 `encrypted:` 값으로 기록해 레포와 로컬 파일의 평문 secret 노출을 막는 용도다.

## 누구에게 팔아야 하나

- API 키와 토큰을 `.env`로 자주 다루는 개발자
- 로컬 개발 중 secret 노출 사고를 줄이고 싶은 팀
- `.env.keys`와 dotenvx 방식의 암호화 플로우를 VSCode 안에서 쓰고 싶은 사람

## 사용법 시나리오

1. `.env.keys`가 있는 폴더에서 `.env` 파일을 연다.
2. Reopen Editor With에서 encrypted import-only 에디터를 선택한다.
3. 키 목록을 확인하고 값 입력칸에는 붙여넣기만 한다.
4. 저장하면 값은 암호화되어 `.env`에 기록된다.
5. 필요하면 암호화 전략을 선택하거나 허브에서 상태를 새로고침한다.

## 썸네일 제작 지시

- 실제 그릴 장면: `.env` 테이블에서 `API_KEY`, `DATABASE_URL` 키는 보이고 값 칸은 마스킹되어 있으며, 아래 파일에는 `encrypted:BASE64...` 문자열이 저장되는 장면.
- 반드시 보여야 할 목적: "secret을 붙여넣고 암호화 저장한다"가 보여야 한다.
- 시각 요소: 마스킹된 입력칸, 자물쇠가 붙은 디스크 파일, `.env.keys`, 암호화 전환 화살표.
- 피할 것: 자물쇠 SVG 하나만 크게 배치, 해커풍 추상 배경, 실제 `.env` 흐름이 없는 이미지.
