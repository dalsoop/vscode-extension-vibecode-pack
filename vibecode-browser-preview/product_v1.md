# Vibecode Browser Preview

> **VSCode 안에서 HTML 더블클릭 → 라이브 프리뷰.**
> 저장하면 즉시 리로드. dev server 안 띄워도 됨.

---

## 누구를 위한 거?

- 디자이너에게 받은 **정적 HTML 시안** 빨리 확인해야 하는 프론트 개발자
- LP / 메일 템플릿 / 출력용 PDF 변환 전 HTML 을 자주 만지는 사람
- `live-server` 매번 따로 띄우기 귀찮은 사람
- 다른 사람 빌드 결과물 (`dist/index.html`) 을 한 번 클릭으로 보고 싶은 리뷰어

## 풀어주는 문제

| 평소 흔한 상황 | 이 확장으로 |
|---|---|
| HTML 파일 더블클릭 → 텍스트 에디터만 뜸 | **더블클릭 = 라이브 프리뷰** (customEditor priority: default) |
| `python -m http.server` 따로 띄우기 | 자동으로 `127.0.0.1:<랜덤포트>` 서버 띄움, 닫으면 자동 종료 |
| 편집해도 매번 새로고침 눌러야 함 | 워크스페이스 어디든 저장하면 **200ms 디바운스 후 자동 리로드** |
| 외부 브라우저로 열어서 비교하고 싶음 | ↗ 버튼 한 번으로 기본 브라우저에서 같은 URL 오픈 |

## 핵심 셀링 포인트

1. **더블클릭 = 끝** — customEditor 로 등록돼서 HTML 클릭 = 프리뷰. 설정 0.
2. **워크스페이스 라이브 리로드** — HTML 뿐 아니라 CSS/JS/이미지 어떤 파일이 바뀌어도 reload.
3. **로컬 HTTP 서버 자동 관리** — 포트 자동 할당, 마지막 탭 닫히면 서버도 죽음.
4. **toolbar 3버튼** — ↻ Reload / ▤ Edit Source / ↗ External Browser. 군더더기 없음.
5. **127.0.0.1 only** — 같은 머신 안에서만 동작. 옆자리에 안 새어나감.

## 사용 흐름 (시각)

```
Explorer  ──────────  Preview Pane (Beside)
                     ┌─────────────────────────────┐
                     │ ↻  ▤  ↗  127.0.0.1:54321  │  ← toolbar
                     ├─────────────────────────────┤
▸ src/                                              │
  ■ index.html ◄── 더블클릭                          │
  ■ style.css                  <iframe src=...>     │
  ■ app.js                                          │
                     │   (렌더된 HTML 페이지)        │
                     │                              │
                     └─────────────────────────────┘
            ↑
   파일 저장 → 200ms → iframe 자동 reload
```

## 썸네일 아이디어

- **분할 화면**: 왼쪽에 HTML 코드, 오른쪽에 렌더된 페이지 — 가운데 큰 **↻ 화살표** 가 양쪽을 잇는 구도
- **포인트 글자**: `Save = Reload`
- **컬러 키**: VSCode 다크 배경 + 노란/시안 강조 (저장 → 리로드의 펄스)
- **부제 그림**: 파일 아이콘 `.html` 위에 더블클릭 커서 — 그 옆에 페이지 미리보기 카드가 슬라이드인
