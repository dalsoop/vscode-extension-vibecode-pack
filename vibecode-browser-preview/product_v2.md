# Product V2 - Vibecode Browser Preview

> 셀링 전용 문서. 개발 README가 아니라 마켓플레이스 설명, 썸네일 기획, 소개 이미지 제작에 쓰는 메시지다.

## 한 줄 판매 문구

HTML 파일을 더블클릭하면 VSCode 안에서 바로 열리고, 저장할 때마다 자동 새로고침되는 로컬 브라우저 프리뷰.

## 어떤 목적으로 쓰나

정적 HTML, 빌드 결과물, 작은 퍼블리싱 파일을 확인하려고 매번 브라우저를 열고 새로고침하는 흐름을 줄인다. 워크스페이스 루트 기준 로컬 서버로 안전하게 띄우고, 수정 즉시 iframe 프리뷰를 갱신하는 용도다.

## 누구에게 팔아야 하나

- HTML/CSS를 빠르게 만지는 퍼블리셔
- 단일 HTML 산출물을 자주 확인하는 개발자
- 브라우저 탭 전환 없이 VSCode 안에서 결과를 보고 싶은 사람

## 사용법 시나리오

1. 워크스페이스 안의 `.html` 또는 `.htm` 파일을 연다.
2. Vibecode Browser Preview 커스텀 에디터가 프리뷰를 띄운다.
3. Edit Source로 원본을 옆에 열어 수정한다.
4. 저장하면 프리뷰가 자동 새로고침된다.
5. 필요하면 External Browser 버튼으로 같은 로컬 URL을 브라우저에서 연다.

## 썸네일 제작 지시

- 실제 그릴 장면: VSCode 화면이 좌우로 나뉘어 왼쪽에는 HTML 코드, 오른쪽에는 실제 웹페이지 프리뷰 iframe이 보이는 장면.
- 반드시 보여야 할 목적: "HTML을 저장하면 VSCode 안 프리뷰가 바로 바뀐다"가 보여야 한다.
- 시각 요소: HTML 코드 줄, 프리뷰 웹페이지, Reload 버튼, 저장 후 반짝이는 변경 지점.
- 피할 것: SVG 브라우저 아이콘만 크게 배치, 추상 그라데이션, 실제 프리뷰 없이 코드만 있는 이미지.
